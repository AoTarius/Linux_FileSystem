# 课程设计要求

## 一、实验目的

1. 通过一个简单多用户文件系统的设计，加深理解文件系统的内部功能和内部实现。
2. 结合数据结构、程序设计、计算机原理等课程的知识，设计一个多级目录文件系统，进一步理解操作系统。
3. 通过对实际问题的分析、设计、编程实现，提高学生实际应用、编程的能力。

---

## 二、功能要求（模仿 Linux 的 EXT2 文件系统）

### 2.1 基础要求

| 序号 | 要求 | 说明 |
|------|------|------|
| 1 | 理解 EXT2 原理 | 理解和掌握 EXT2 文件系统的原理和实现 |
| 2 | 模拟磁盘 | 创建模拟文件系统，使用二进制文件模拟磁盘空间 |
| 3 | 信息存储 | 存储所有用户信息、inode 信息、超级块信息、文件信息 |
| 4 | 用户交互 | 支持用户交互，包括 **login 用户登录** |
| 5 | 安全性 | 源文件可以进行读写保护（读写权限） |

## 三、磁盘结构要求

模仿 EXT2 文件系统的磁盘布局：

```
┌─────────────┬──────────────────┬──────────────┬──────────────┬──────────────┬─────────────┐
│ Superblock  │ Group Desc Table │ Block Bitmap │ Inode Bitmap │ Inode Table  │ Data Blocks │
│   (超级块)   │    (组描述符)      │   (块位图)    │  (inode位图)  │  (inode表)   │   (数据块)   │
└─────────────┴──────────────────┴──────────────┴──────────────┴──────────────┴─────────────┘
```

### 3.1 各区域说明

| 区域 | 存储内容 |
|------|----------|
| **Superblock**（超级块） | 文件系统结构信息：block 和 inode 的总量、未使用的 block 和 inode 数量、block 和 inode 的大小等 |
| **Group Description Table**（组描述符） | 描述每个组的开始与结束块号码，说明每个块分别介于哪个 block 号码之间 |
| **Block Bitmap**（块位图） | 记录哪个数据块已被占用、哪个数据块未被占用 |
| **Inode Bitmap**（inode 位图） | 记录使用与未使用的 inode 号 |
| **Inode Table**（inode 表） | 存放所有文件的 inode 信息 |
| **Data Blocks**（数据块） | 存放文件内容 |

---

## 四、Inode 结构要求

每个 inode 节点中必须包含以下信息：

| 字段 | 说明 |
|------|------|
| 文件权限 | 文件的读写权限（rwx） |
| 拥有者/组 | 文件的拥有者和所属组（owner / group） |
| 文件容量 | 文件的大小 |
| ctime | 文件的创建时间 |
| atime | 文件最近一次的读取时间 |
| mtime | 文件最近修改的时间 |
| 数据块指针 | **12 个直接块 + 1 个间接块 + 1 个双间接块 + 1 个三间接块** |
